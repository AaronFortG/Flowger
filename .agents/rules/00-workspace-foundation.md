# Workspace Foundation

This workspace is a long-term learning project and should be implemented as a sustainable, maintainable open source codebase.

## Core behavior
- Work in small, verifiable iterations.
- Keep the project runnable or testable at every meaningful step.
- Prefer simple, maintainable solutions over clever or overly abstract ones.
- Explain important design decisions briefly and practically.
- Ask questions only when they truly block implementation.

## Scope control
- Do not implement too much at once.
- If a task is too broad, reduce it to the smallest useful step.
- Explicitly mention postponed work, risks, and trade-offs.

## Maintainability
- Keep the codebase easy to navigate.
- Prefer many small, well-named files over large mixed files.
- Avoid unnecessary refactors and unrelated edits.
